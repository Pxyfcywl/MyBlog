## 三、MinerU 文档解析 API

PDF/Office/图片 → Markdown，使用精准解析 API 的批量上传接口。

### 基本信息

| 项目 | 值 |
|------|-----|
| Base URL | `https://mineru.net/api/v4` |
| 认证 | `Authorization: Bearer {token}` |
| Token 获取 | MinerU 后台 → API 管理 → 创建 |
| 模型版本 | `vlm`（推荐）/ `pipeline` / `MinerU-HTML` |
| 文件限制 | ≤ 200MB，≤ 200 页 |
| 每日额度 | 1000 页高优先级，超出降速 |

### 调用流程（4 步）

```
① POST /file-urls/batch            申请上传链接 → batch_id + file_urls
② PUT  {file_url}                  上传文件（无需 Header）
③ GET  /extract-results/batch/id   轮询结果（每 3 秒）→ state=done
④ 下载 zip → 解压 full.md          拿到 Markdown
```

### ① 申请上传链接

```
POST https://mineru.net/api/v4/file-urls/batch
Header: Authorization: Bearer {token}
```

```json
{
  "files": [
    {"name": "文件1.pdf"},
    {"name": "文件2.pdf"}
  ],
  "model_version": "vlm",
  "is_ocr": true,
  "enable_formula": true,
  "enable_table": true
}
```

返回 `batch_id` 和 `file_urls`（与 files 一一对应）。

### ② 上传文件

```
PUT {file_url}
Body: 文件二进制（无需 Header）
```

上传完成后系统**自动提交**解析任务。链接有效期 24 小时。

### ③ 轮询结果

```
GET https://mineru.net/api/v4/extract-results/batch/{batch_id}
```

state 状态：`waiting-file` → `pending` → `running` → `done` / `failed`

### ④ 获取 Markdown

```python
import io, zipfile, httpx

resp = httpx.get(full_zip_url)
zf = zipfile.ZipFile(io.BytesIO(resp.content))
markdown = zf.read("full.md").decode("utf-8")
```

zip 内容：`full.md`（Markdown）、`*_content_list.json`、`*_model.json`、`*_middle.json`

### 错误码

| 错误码 | 含义 | 处理 |
|--------|------|------|
| `A0202` | Token 错误 | 检查格式 |
| `A0211` | Token 过期 | 重新生成 |
| `-60005` | 超 200MB | 拆分文件 |
| `-60006` | 超 200 页 | 拆分文件 |
| `-60018` | 额度用尽 | 等明天 |

### Agent 轻量解析（备用，免 Token）

10MB / 20 页，无需认证，IP 限频。结果直接返回 Markdown CDN 链接，不需要解压。

```
POST /api/v1/agent/parse/file    → task_id + file_url
PUT  {file_url}                  → 上传文件
GET  /api/v1/agent/parse/{task_id} → markdown_url
```
